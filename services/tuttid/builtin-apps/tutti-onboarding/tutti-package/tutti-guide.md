# Tutti 产品知识库

> 本文档供 AI Agent 在回答用户关于 Tutti 产品的问题时作为参考。请基于以下内容准确回答，不要编造本文档未提及的功能。

## 产品定义

- **产品名称：** Tutti
- **品牌定位：** Agent OS —— 实时共享的工作空间
- **一句话介绍：** Tutti 是人与 Agent「同频」创造的地方，上下文、文件、应用、任务，全部打通。
- **Slogan（英文）：** Where people and agents build in tune.
- **核心价值：** Agent 不再各做各的，而是在同一个实时工作空间里共享上下文、文件、应用和任务。你的 Codex 能看到 Claude 改了什么、正在运行什么、项目当前处于什么状态。
- **面向用户：** 任何用 AI Agent 来 build 的人——独立开发者、设计师、产品经理、内容创作者。
- **使用门槛：** 全 GUI 界面，无需终端命令行，打开就能用。

## 核心能力

### 1. 连接 Agent（Bring Your Own Subscriptions）

- 一键连接你本地的 Agent，复用你已有的订阅，零额外费用。
- **当前支持：** Claude Code、Codex。
- **即将支持：** OpenClaw、Gemini、Hermes（Kimi）。
- 无需终端，上手简单，不同 Agent 间交互体验一致。

> 当用户问"支持哪些 Agent"时：回答当前支持 Claude Code 和 Codex，OpenClaw、Gemini、Hermes 正在开发接入中。

> 当用户问"需要额外付费吗"时：回答 Tutti 复用你已有的 Agent 订阅，不需要额外付费。可使用的应用能力取决于你已有的 Agent / 模型订阅。

### 2. Big @ 协作（实时共享工作空间）

这是 Tutti 的核心差异化能力。Agent 之间不是简单交接摘要，而是共享同一个实时工作空间。

**@ 的四种用法：**

| 用法   | 操作                    | 效果                                 |
| ------ | ----------------------- | ------------------------------------ |
| @ 会话 | 在对话中 @ 另一个 Agent | 共享会话上下文，无需重新解释项目背景 |
| @ 文件 | 在对话中 @ 文件         | Agent 直接读取文件内容               |
| @ 任务 | 在对话中 @ 任务         | Agent 获取任务上下文                 |
| @ 应用 | 在对话中 @ 应用         | 让 Agent 调用应用能力完成任务        |

**引用 "+"：** 在 Agent 对话框点击 "+"，可以引用本地文件、引用应用生成的产物。

**与竞品的区别：** 其他产品只做到了"消息层共享"（像一个群聊），Agent 在频道里说话，但看不到彼此的文件和进程。Tutti 提供的是实时工作空间级别的共享。

> 当用户问"怎么让两个 Agent 协作"时：回答在对话中使用 @ 即可。例如在 Codex 中 @ Claude Code 的历史对话，Codex 就能直接在 Claude 已有的工作基础上继续构建，无需重新解释上下文。

### 3. 内置应用（The Apps）

Tutti 内置应用市场，你能用，Agent 也能调用。所有应用均复用你本地的 Agent 订阅。

**当前可用应用：**

| 应用         | 能力                                                               |
| ------------ | ------------------------------------------------------------------ |
| AI Canvas    | 生成图片、生成视频（类似 Canvas / Lovart）                         |
| 原型设计     | 生成 UI/UX 原型设计稿（类似 Claude Design）                        |
| AI 文档      | 起草、改写和整理文档，支持 HTML、Markdown、连接本地 Word，丰富模版 |
| AI PPT       | AI 生成 PPT、HTML，支持单页手动编辑文案、排版                      |
| 群聊         | 与多个 Agent 同时协作对话                                          |
| 自动化任务   | 创建和调度自动化任务                                               |
| 每日产品雷达 | 汇总每日新产品和趋势项目                                           |

**两种使用方式：**

- **自己用：** 直接在应用中心打开应用使用。
- **Agent 用：** Agent 在对话中 @ 应用直接完成任务，也能拿应用的产物继续工作，不需要你手动搬运。

**关于应用能力与订阅的关系：** Tutti 复用你已配置好的 Agent 能力。例如，如果你在 OpenClaw 中配置了支持生图的模型，就可以在 Tutti 里使用生图应用。Codex 模型本身具备生图能力，所以也可以通过 Codex 调用生图应用。但如果某个 Agent 的订阅本身不支持某项能力（如 Claude Code 订阅不支持生图），就无法通过该订阅使用对应应用。

> 当用户问"应用需要单独付费吗"时：回答不需要，所有应用复用你已有的 Agent 订阅。可使用的应用能力取决于你的 Agent / 模型订阅所支持的能力。

### 4. 目标拆解与任务管控

从目标到任务完成，全程掌控。你定目标，Agent 拆解执行，全程可控，异常不漏。

**目标拆解（Issue Center）：**

- 你只需描述目标，Tutti 自动拆解为清晰的子任务。
- 你审核每个任务，分配给合适的 Agent 执行。
- 不同 Agent 擅长的能力不一样，你可以把合适的任务分配给合适的 Agent。
- 各 Agent 彼此"可见"，可以自动回避或处理冲突，自己判断该并行还是串行。

**任务管控（Control Center）：**

- 一个视图看全局：所有 Agent 对话状态、待审批的操作、正在运行的任务。
- 需要你确认的地方，快速定位一键批。
- 不用再在多个 Tab 中来回切换。

> 当用户问"怎么给多个 Agent 分配任务"时：回答在任务中心描述你的目标，Tutti 会自动拆解为子任务，你可以把每个子任务分配给不同的 Agent 并行执行。

### 5. 个性化设置

- 深色模式 / 浅色模式
- 更换桌面背景
- 修改程序坞位置
- 自定义图标样式

## 版本说明

### 本地版（当前可用，开源）

Agent 跑在本地，工作态在本地。核心能力：

- 跨 Agent 上下文共享（@ 协作）
- 内置应用生态
- 复用已有 Agent 订阅
- 目标拆解与任务管控

### Tutti · VM（即将推出，可在官网加入 Waitlist）

包含本地版全部能力，额外实现：

- **跨设备协作：** 开个云端空间，多台设备在里面工作，就像在用同一台电脑。
- **多人协作：** 同一个云端工作空间内，可以看到彼此的对话、文件、产物、任务进展。
- **跨人引用：** @ 引用同事的文件、与 Agent 的对话，并让你的 Agent 在此基础上构建。
- **Localhost 预览共享：** 本地跑起来的网站，朋友可以在云端工作空间直接打开预览。
- **跨人任务分配：** 将任务分配给同事的 Agent 执行。

> 隐私说明：共享仅针对以房间为维度的共享，邀请人与受邀人需加入同一房间，在同一房间产出的内容才会被共享。

## 常见问题

**Q: Tutti 是什么？**
A: Tutti 是一个 Agent OS——实时共享的工作空间。让你的多个 AI Agent 在同一个工作空间里共享上下文、文件、应用和任务，协同工作。

**Q: Tutti 免费吗？**
A: Tutti 本地版是开源免费的。它复用你已有的 Agent 订阅（如 Claude、Codex），不需要额外付费。

**Q: 需要会写代码才能用吗？**
A: 不需要。Tutti 是全 GUI 界面，无需终端命令行，打开就能用。

**Q: 支持哪些 Agent？**
A: 当前支持 Claude Code 和 Codex。OpenClaw、Gemini、Hermes 正在开发接入中。

**Q: 应用需要单独付费吗？**
A: 不需要。所有内置应用复用你已有的 Agent 订阅。可使用的应用能力取决于你的 Agent / 模型订阅所支持的能力。

**Q: Tutti 和其他多 Agent 产品有什么区别？**
A: 其他产品大多只做到"消息层共享"——Agent 在频道里说话，但看不到彼此的文件和进程。Tutti 提供的是实时工作空间级别的共享：上下文、文件、应用、任务，全部打通。

**Q: Tutti · VM 什么时候上线？**
A: Tutti · VM 即将推出，可以在官网 www.tutti.sh 加入 Waitlist。

**Q: 我的数据安全吗？**
A: 本地版的数据全部在你本地。Tutti · VM 的共享仅以房间为维度，只有加入同一房间的人才能看到房间内的内容。
